# Первый сценарий

Создайте сценарий в интерфейсе администрирования или импортируйте YAML-описание.
Сценарию нужны уникальный slug, отображаемое имя, хотя бы один шаг и настроенный
для этого шага драйвер.

Полное описание редактируется в YAML-редакторе. Перед публикацией новой неизменяемой
версии форма проверяет документ.

Поддерживаемые поля перечислены в [справочнике синтаксиса сценариев](/ru/reference/scenario-syntax).
