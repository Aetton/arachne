# Сценарии

Опубликованные сценарии хранятся в PostgreSQL. Файл `config/scenarios.yaml` служит
только начальным набором данных: slug импортируется, лишь если его ещё нет в базе.

При редактировании сценария создаётся новая версия. У существующих запусков сохраняются
ID версии и неизменяемый снимок описания, поэтому последующие изменения не переписывают
историю выполнения.

Доступ к сценарию определяется возможностями, ролями, командами и ACL самого сценария.
