# Security Policy

## Reporting vulnerabilities

Please do not report security issues through public GitHub issues.

Use the private reporting channel configured for this repository, or contact the maintainers privately.
Include:

- affected version or commit;
- deployment mode;
- impact;
- reproduction steps;
- relevant logs with secrets removed.

## Sensitive data

Do not include secrets, private keys, access tokens, internal hostnames, customer data, or production logs in issues, pull requests, screenshots, or examples.

Use placeholders such as:

```text
forgejo.example.internal
nexus.example.internal
arachne.example.internal
```

## Supported versions

Arachne is currently pre-1.0 software. Security fixes target the main branch unless a release branch is explicitly announced.
